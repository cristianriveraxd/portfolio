Sobre mí

Que tal, llamo Cristian Rivera. Hace aproximadamente dos años, comencé mi viaje en el mundo del desarrollo de software. Lo que comenzó como una simple curiosidad se ha convertido en una verdadera pasión. A lo largo de este tiempo, he tenido la oportunidad de trabajar en diversos proyectos que me han permitido aprender y aplicar nuevas tecnologías.
Cada proyecto ha sido una experiencia enriquecedora, ayudándome a crecer tanto profesional como personalmente. Mi entusiasmo por la tecnología me impulsa a estar en constante aprendizaje, explorando y adoptando las últimas tendencias y herramientas del sector.
Soy un firme creyente en el poder transformador de la tecnología y estoy comprometido en utilizar mis habilidades para crear soluciones innovadoras y eficientes. Me apasiona la idea de que, a través del desarrollo de software, podemos mejorar la vida de las personas y resolver problemas complejos.

¿Pero quien es Cristian?

Cristian es un soñador visionario e innovador apasionado por la tecnología y su aplicación en busca de mejorar la sociedad.
Todo comenzó estudiando mecatrónica en el Servicio Nacional de Aprendizaje, donde realice una tecnología en Automatización Industrial, lo que despertó mi interés al ver la mezcla de tecnologías aplicadas en el campo industrial para llevar a cabo un proceso. ¿Pero como un futuro mecatronico termino estudiando desarrollo de software? Durante mis pasantías, descubri el impacto del desarrollo de software, en el control industrial, al ver cómo una aplicación podia manejar sistemas robóticos que mueven cargas de 700 kg, el como se registra, procesa y se muestra la información crucial del proceso, el como a través de reconocimientos faciales se daba acceso a determinadas areas, ahí fue cuando se desperto la curiosidad por este mundo tan grande.
Actualmente, curso 7mo semestre de Ingeniería de Sistemas en la ETITC y lidero un proceso de paletizado automático en una empresa de alimentos. Además de mi trabajo y compromisos estudiantiles, intento llevar una vida equilibrada con pasatiempos como el atletismo, aprender sobre fotografía y el mundo del deporte.

Proyectos

Sistema de Monitoreo
Este proyecto consistió en extraer y mostrar información de un sistema de paletizado automático utilizando PHP y MySQL. El objetivo principal fue mejorar la accesibilidad y disponibilidad de los datos del proceso, ya que la información estaba centralizada en un grupo reducido de personas, lo que generaba problemas críticos, como la pérdida de información, datos incompletos, y errores en la interpretación. Además, se presentaban fallas frecuentes en las unidades de almacenamiento, que afectaban tanto la integridad de la información en tiempo real. Estos inconvenientes impactaban directamente en la eficiencia del sistema, ocasionando problemas operativos. La implementación del proyecto permitió solucionar estos inconvenientes al automatizar el acceso a los datos y mejorar la gestión de la información en todo el sistema, garantizando su disponibilidad y reduciendo la posibilidad de errores humanos.

Formula 1 - Tienda
Este proyecto fue parte de mi materia de Programación III y consistió en el desarrollo de una aplicación web. El front-end se implementó utilizando solo CSS, mientras que el back-end se desarrolló con tecnologías como Servlets, JSP, JSTL, JavaBeans, MySQL y Apache TomEE Plume como servidor. También se utilizó JavaScript para agregar interactividad en el front-end.
El proyecto se trabajó en equipo bajo la metodología ágil Scrum, y se utilizó Git como controlador de versiones para la colaboración. Mi rol en el proyecto fue desarrollar la tienda, tanto el front-end como la lógica del negocio. Fue un reto interesante ya que la mayor parte del desarrollo se realizó en un entorno Linux utilizando la consola y el servidor Apache TomEE Plume corriendo en Linux por consola. Por tiempo el proyecto no alcanzo a ser desplegado, en el enlace al proyecto redirige a una página estática desplegada en GitHub Pages, pero el codigo lo pueden encontrar en el repositorio.

Modulo de gestión de flujos - SST
Este proyecto consistió en un desafío empresarial para desarrollar un módulo de gestión de flujos de trabajo en una aplicación web orientada a la gestión de permisos de trabajos en alturas. La aplicación se desarrolló utilizando una variedad de tecnologías, cada una con un rol específico en el proyecto:
- Flask: Microframework utilizado para el back-end de la aplicación. Proporcionó las herramientas necesarias para crear rutas, manejar solicitudes y gestionar la lógica de manera eficiente
- Bootstrap: Utilizado para el front-end facilitó la creación de la interfaz, haciéndola atractiva y responsiva
- CSS nativo: Se empleó para estilizar la aplicación y proporcionar una apariencia personalizada que complementara el diseño de Bootstrap.
- MySQLServer: Servidor de base de datos utilizado para almacenar y gestionar la información de la aplicación, como datos de usuarios, flujos de trabajo y registros de actividades.
Algunas otras bibliotecas complementarias como Flask-Login para asegurar las autenticaciones de usuarios, importlib-metada, itsdangerous para asegurar versiones y tokens de ingreso, jinja2 para renderizar vistas, MarkupSafe para proporcionar un escape seguro para los datos HTML, evitando vulnerabilidades como la inyección de código en las plantillas Jinja2, Werkzeug se utilizó junto con Flask para gestionar solicitudes y respuestas HTTP, y manejar aspectos del servidor web y zipp para manejar archivos comprimidos en el proyecto.
Este proyecto representó un desafío significativo en el desarrollo de mi carrera, ya que involucró la aplicación de todas las habilidades adquiridas durante el primer ciclo de estudios. El proyecto fue desplegado de manera local, pero puedes encontrar más información y una simulación detallada en el repositorio.
